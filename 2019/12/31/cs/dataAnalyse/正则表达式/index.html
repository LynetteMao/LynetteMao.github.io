字符匹配——day3

1、BeautifulSoup

官方文档

1、用法

r=requests.get(url,verify=False)
bsObj =  BeautifulSoup(r)
print(bsObj)  

##报错   
gives the error message "TypeError: object of type 'Response' has no len().

requests.get(url)返回的是一个response对象，而BeautifulSoup需要传入的是一个str类型的数据，所以需要把response对象转化为string对象，加上r.text再传入BeautifulSoup(r.text)，且可以指定解析器(lxml、html5lib等，详见官方文档)。
要让格式好看，可以加上bsObj.prettify()

2、基本语法

四大基本对象:Tag\Navigablestring\BeautifulSoup\Comment
Tag有Name和Attributes两个属性，tag.name获取tag的名称，
获取tag的属性：
一个tag可能有很多个属性. tag 有一个 “class” 的属性,值为 “boldest” . tag的属性的操作方法与字典相同:

tag['class']
# u'boldest'
也可以直接”点”取属性, 比如: .attrs :

tag.attrs
# {u'class': u'boldest'}
tag的属性可以被添加,删除或修改. 再说一次, tag的属性操作方法与字典一样

tag['class'] = 'verybold'
tag['id'] = 1
tag
# <blockquote class="verybold" id="1">Extremely bold</blockquote>

del tag['class']
del tag['id']
tag
# <blockquote>Extremely bold</blockquote>

tag['class']
# KeyError: 'class'
print(tag.get('class'))
# None

获取tag的字符串，其type是NavigableStirng对象的，此对象是继承了String类，其用法其实差不多。

unicode_string = unicode(tag.string)    
#替代其中的字符串  
tag.string.replace_with("No longer bold")

3、遍历

  1. contents && children
    contents和children都能够获取tag下的直接子节点，contents返回所有子节点的列表，children返回所有子节点的迭代器。

    迭代器：不预先分配好内存进行存储，存放的是算法，在调用next()才计算出下一个数据。所以children读值需要for i in children

  2. strings && stripped_strings

    • string：获取某个标签下的非标签字符串。返回来的是个字符串。如果这个标签下有多行字符，那么就不能获取到了。
    • strings：获取某个标签下的子孙非标签字符串。返回来的是个生成器。
    • stripped_strings：获取某个标签下的子孙非标签字符串，会去掉空白字符。返回来的是个生成器。
    • get_text：获取某个标签下的子孙非标签字符串，以普通字符串形式返回

2、正则表达式

正则表达式

零宽度断言

零宽度断言指的是不占有内存的匹配，即只匹配位置。
作用：匹配具体格式的位置，一般前后会加上匹配的正则表达式，输入的断言不进行匹配。通俗来说，就是匹配特定格式前或者后的字符串。
?=和?!相对，?=匹配某格式前面的内容，?!除某格式外前面的内容。
?<=和?<!相对，匹配某格式后面的内容。

贪婪匹配和惰性匹配

正则表达式默认使用贪婪匹配，即会尽可能多的匹配字符，就会出现很多不需要的字符。
加上?进行惰性匹配，会在匹配到第一个符合条件的字符就停止匹配。

分组

()表示捕获组，会对每一个捕获组分配一个编号，0表示整个获取的字符串，1表示第一个捕获组，以此类推。可以通过反向引用，对分组已捕获的字符串进行引用。
